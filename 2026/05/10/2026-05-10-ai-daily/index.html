Codex CLI v0.130.0：远程控制与插件生态的里程碑

背景：为什么这个话题重要

Codex CLI 在 v0.130.0 版本中带来了三个关键更新：codex remote-control 远程控制入口、插件详情展示 bundled hooks、以及插件分享的可发现性控制。这标志着 OpenAI 的编码工具从”本地助手”向”可编排的分布式 Agent 基础设施”演进。对于用 Codex 做自动化部署、CI/CD 集成、或团队共享工作流的开发者来说，这次更新直接解决了”怎么在服务器上跑 Codex”和”怎么安全地共享团队插件”两个核心痛点。

原理：技术实现的核心逻辑

远程控制架构：新增的 codex remote-control 是一个 headless app-server 的简化入口。传统 Codex 是交互式 TUI 工具，而这次更新让它可以作为后台服务启动，接受远程客户端的指令。这意味着你可以在一台 GPU 服务器上常驻 Codex 进程，通过 API 或 WebSocket 从其他机器（甚至移动端）发送编码任务。

插件元数据暴露：插件现在可以声明自己捆绑了哪些 hooks（生命周期钩子），这让插件市场在安装前就能展示能力边界。配合”discoverability controls”，插件作者可以控制插件的可见范围——从完全公开到仅限组织内部。

客户端能力扩展：App-server 客户端现在能执行更完整的操作序列，包括文件系统访问、沙箱命令执行、以及结果回传。这实际上构建了一个”Codex 即服务”的最小可行架构。
